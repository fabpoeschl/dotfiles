  <F12>             Step out

  <leader>dl        Log point (prints message without stopping)

  <leader>di        Toggle DAP UI panels

  <leader>dh        Hover variable under cursor
  <leader>do        Toggle REPL

Debug Configurations ~

  Python:
    Launch file                         Run current .py file
    Launch with arguments               Prompts for CLI arguments
    Attach remote (localhost:5678)      For kubectl port-forward / debugpy
    Attach remote (custom host:port)    Prompts for host and port

  JavaScript / TypeScript:
    Launch file (Node)                  Run current file with Node
    Attach to Node (localhost:9229)     For --inspect / kubectl port-forward
    Attach to Node (custom host:port)   Prompts for host and port
    Launch Chrome                       Debug in Chrome (prompts for URL)

  Ruby:
    Launch file                         Run current .rb file
    Launch with bundler                 bundle exec ruby <file>
    Launch Rails server                 bundle exec rails server
    Launch RSpec (current file)         bundle exec rspec <file>
    Attach remote (localhost:1234)      For rdbg / kubectl port-forward
    Attach remote (custom host:port)    Prompts for host and port

Remote Debugging (Kubernetes) ~

  The "Attach remote" configurations connect to a debug server over TCP.
  Use kubectl port-forward to bridge the pod's debug port to localhost:

    # Python (debugpy)
    kubectl port-forward pod/my-app 5678:5678
    # Then: <F5> -> "Attach remote (localhost:5678)"

    # JavaScript / Node (--inspect)
    kubectl port-forward pod/my-app 9229:9229
    # Then: <F5> -> "Attach to Node (localhost:9229)"

    # Ruby (rdbg / debug.gem)
    kubectl port-forward pod/my-app 1234:1234
    # Then: <F5> -> "Attach remote (localhost:1234)"

  On the pod, start the app with its debug server enabled:

    # Python:  python -m debugpy --listen 0.0.0.0:5678 --wait-for-client app.py
    # Node:    node --inspect=0.0.0.0:9229 app.js
    # Ruby:    rdbg --open --host 0.0.0.0 --port 1234 -c -- ruby app.rb

Per-project remote workspace root ~

  The "Attach remote" Python and Node configs default to /app for the
  remote workspace root (where source files live inside the container).
  Override per project by creating a .nvim.lua at the project root:

    -- .nvim.lua
    vim.g.dap_remote_root = "/srv/app"

  Then run :trust on first open to authorize the file. exrc is enabled
  globally (see options.lua), so the file is auto-sourced from cwd.

DATABASE (vim-dadbod)                                *cheatsheet-database*

Connection Management ~
  <leader>Du        Toggle DBUI drawer
  <leader>Da        Add a connection manually
  <leader>Dc        Connect to remote DB (K8s pod)
  <leader>Dd        Disconnect remote DB

Remote Connect (:DBConnect) ~
  :DBConnect -c <ctx> -n <ns> -d <workload>
  :DBConnect -c <ctx> -n <ns> -d <workload> -p 5433 -l 15432 -b mydb -u admin

  Flags:
    -c, --context     Kubernetes context
    -n, --namespace   Kubernetes namespace
    -d, --database    Deployment / StatefulSet name (resolved via ownerRefs)
    -u, --user        DB user (default: $DBCONNECT_USER or "postgres")
    -p, --port        Remote port (auto-detected from name if omitted)
    -l, --local-port  Local port to forward to (default: same as remote)
    -b, --dbname      Database name in URL (defaults to -d value)

  :DBDisconnect [name]  Stop port-forward and remove connection
                        Tab-completes active connection names; with no
                        argument prompts via vim.ui.select.

Credentials ~
  Set $DBCONNECT_CREDENTIALS_CMD to a script that prints either:
    password                   (1 line, uses default user)
    username\npassword         (2 lines, supplies both)
  The script receives -n <ns> -d <database> -c <context>.
  If unset, :DBConnect prompts via inputsecret().

Local Databases ~
  Use <leader>Da to add a local connection URL directly:
    postgres://user:pass@localhost:5432/mydb
    mysql://root@localhost:3306/mydb
    sqlite:path/to/db.sqlite

KUBERNETES                                        *cheatsheet-kubernetes*

Pod Connection (:PodConnect) ~
  <leader>ks        Shell into pod (prompts for context/ns/app)
  <leader>kl        Tail pod logs
  <leader>kf        Port-forward pod

  :PodConnect -c <ctx> -n <ns> -a <app>                 Shell (default)
  :PodConnect -c <ctx> -n <ns> -a <app> logs             Tail logs
  :PodConnect -c <ctx> -n <ns> -a <app> forward 3000:3000

  All actions open in a terminal split. Close the buffer to stop.

Pod matching ~
  -a <app> resolves to a Deployment / StatefulSet / DaemonSet / Job by
  inspecting each pod's ownerReferences (for ReplicaSet owners the trailing
  pod-template-hash is stripped to get the Deployment name).

Shell Scripts (~/.local/bin/) ~
  remote-db-connect -c <ctx> -n <ns> -d <workload> [-p <port>] [-l <local-port>]
  remote-pod-connect -c <ctx> -n <ns> -a <workload> [shell|forward|logs]
